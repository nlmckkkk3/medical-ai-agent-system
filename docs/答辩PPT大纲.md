## Slide 2: 项目背景与动机
人们面临的健康消费困惑：
- 怎么选体检套餐？哪个药更适合？
- 报告指标什么意思？症状该挂哪个科？
AI 技术的发展为解决这些问题提供了新可能
目标：为普通用户提供可靠的健康消费决策辅助

## Slide 3: 功能概览
四大核心功能模块：
1. 体检套餐选择（年龄/性别/预算匹配）
2. 药品/保健品对比（结构化对比表格）
3. 体检报告解读（多模态输入）
4. 就医科室推荐（症状分析匹配）
展示界面截图（可使用 screenshots/ 目录下的图片）

## Slide 4: 技术架构
4层 Agent Pipeline：
Safety Agent → Router Agent → Business Agent → Reflection Agent
知识库：ChromaDB + BGE Embedding / 关键词双模式
多模态输入：文本 + 图片(OCR) + PDF
用户记忆：JSON 文件持久化

## Slide 5: 设计与实现亮点
1. 安全性设计：关键词预过滤 + LLM 深度检测双层安全
2. 质量保障：Reflection Agent 5维度质量审核（边界/一致性/安全/免责/完整性）
3. 用户体验：场景专属加载提示 + 一键示例问题 + 多轮对话

## Slide 6: 安全性与合规性
- 不做诊断，不提供治疗方案
- 所有输出标注“仅供参考”
- 用户数据仅存储在本地
- 明确免责声明，位置醒目
